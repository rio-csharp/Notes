# .NET + React Full-stack Engineering Knowledge Base

This knowledge base is a practical learning resource for full-stack engineering with `.NET + React`, plus architecture, security, performance, data systems, and production operations.

The goal is not to memorize definitions. The goal is to understand how systems work, write practical code, reason about trade-offs, debug real problems, and explain technical ideas clearly.

Key Chinese notes are included for difficult terms, for example `idempotency（幂等性）` and `eventual consistency（最终一致性）`.

## How To Use This Knowledge Base

1. Start with [06-30-day-study-plan.md](./06-30-day-study-plan.md) if you want a daily learning plan.
2. Use [02-learning-path.md](./02-learning-path.md) if you want the high-level learning order.
3. Use [05-fullstack-engineering-checklist.md](./05-fullstack-engineering-checklist.md) to measure practical readiness.
4. Study the core stack first: .NET, C#, ASP.NET Core, DI, EF Core, SQL, TypeScript, React.
5. Then move to architecture, security, performance, distributed systems, and system design.
6. For every topic, practice explaining it in 2 minutes, 5 minutes, and 15 minutes.

## Learning Level

This material is useful for learners who want to grow toward:

- Full-stack Engineer
- Advanced Full-stack Engineer
- .NET Engineer
- React Engineer
- Tech Lead
- Solution Architect
- Application Architect

## Core Capabilities

- Build full features end to end from database to API to UI.
- Understand runtime behavior, not just framework usage.
- Design maintainable, testable, secure services.
- Debug production issues using logs, metrics, traces, and reasoning.
- Discuss trade-offs rather than giving one-size-fits-all answers.
- Communicate clearly with engineers, product managers, QA, DevOps, and leadership.

## Suggested Daily Routine

- 60 minutes: study one technical topic.
- 30 minutes: write or run code examples.
- 30 minutes: answer knowledge checks out loud.
- 20 minutes: review one system design problem.
- 10 minutes: write a short project case note.

## Learning Pattern

For each topic, try to produce:

1. a short definition;
2. why it matters;
3. a practical example;
4. important trade-offs;
5. common mistakes;
6. a real project scenario where the concept appears.
